# 🌌 Autonomous Agent Framework (AAF)

**AAF** - это полноценная OS-level сущность, асинхронный ИИ-агент и мульти-агентная платформа, построенная на микросервисной Event-Driven архитектуре.

В то время как популярные решения делают ставку на легковесность, TypeScript и хранение логов в текстовых файлах, **AAF - это тяжеловесный инженерный фреймворк на Python**. Он создан для тех, кому нужна сложная графовая память, абсолютная изоляция выполняемого кода и настоящая проактивность.

Большинство современных Open-Source агентов страдают от трех проблем: амнезия (забывают контекст через 5 шагов), зацикливание (бесконечные ReAct-петли) и ад зависимостей при установке. **AAF решает их все.**

---
## 🔥 Архитектурные особенности

*   **♾️ Multi-Agent Architecture & CLI Manager:** AAF позволяет развернуть и оркестрировать любое количество независимых агентов на одном сервере. Управление (создание, авторизация, запуск и логирование) происходит через удобный интерактивный терминал (`aaf.py`). Docker Compose собирается автоматически.
*   **🧠 Тройная гибридная память (State-of-the-Art):** в отличие от фреймворков, полагающихся только на векторный поиск, система AAF не теряет контекст. Она комбинирует Vector DB (ChromaDB: семантика), PostgreSQL (жесткие правила и задачи) и полноценный **GraphRAG** на базе KuzuDB (нелинейные ассоциации и интуиция агента).
*   **⚡ Асинхронная шина событий (EventBus):** мозг агента полностью отвязан от сенсоров. Сообщения из Telegram, системные логи и фоновые задачи падают в единую очередь приоритетов. Агент сам решает, на что реагировать прямо сейчас, а что оставить на потом. У агента есть 3 независимых цикла: реакция на события, проактивность и интроспекция.
*   **🐝 Agent Swarm System:** главный агент - оркестратор автономных субагентов. Мозг динамически спавнит специализированных "воркеров" (для OSINT или дебага) и бессмертных "демонов" (для мониторинга), делегируя им рутину для экономии дорогих токенов.
*   **👁️ Мультимодальность:** используйте любую текстовую модель (Gemini, Claude, GPT, GLM) как основной мозг. При получении фото, голосовых сообщений или стикеров система автоматически пропускает медиа через выделенную Vision/Audio-модель (сопроцессор) и возвращает мозгу подробный текст.
*   **💸 Бесплатная работа (API Rotator):** встроенный менеджер ключей использует алгоритм Round-Robin и автоматически переключается на следующий ключ при достижении лимитов (ошибка 429).
*   **🛡️ WatchDog & Self-Healing:** если системный модуль падает, агент просыпается с максимальным приоритетом, читает Traceback и пытается найти решение проблемы.
*   **🧩 Система нативных плагинов (Zero-Boilerplate):** забудьте про ручное написание JSON-схем для OpenAI Tools. Вы просто кладете Python-файл в папку агента, вешаете декоратор `@llm_skill`, и фреймворк сам генерирует схему, валидирует типы, пробрасывает ключи и оборачивает всё в отказоустойчивый асинхронный поток. Зависимости (`custom_requirements.txt`) устанавливаются Docker-ом на лету.

## 🏗 Структура проекта
Проект использует строгий `src-layout` и разделен на ядро и профили агентов:

*   `Agents/` - Профили ваших агентов. Внутри каждого профиля (например, `Agents/VEGA/`):
    *   `.env` - индивидуальные API-ключи агента.
    *   `config/` - индивидуальные настройки и промпты личности.
    *   `workspace/` - изолированная зона баз данных, временных файлов и песочницы (`sandbox`).
*   `src/` - Исходный код логики (Утилиты, Базы данных, Сенсоры, Мозг, Рой). Общий для всех агентов.

## ⚙️ Установка (Quick Start)

Никаких `pip install`, конфликтов версий C++ компиляторов и ручной настройки баз данных. Проект упакован в Docker и разворачивается через встроенный CLI-менеджер.

**Требования:** Установленный Docker Desktop и Python 3.11+ (только для запуска CLI-менеджера на хосте).

### Шаг 1. Клонирование репозитория

```bash
git clone https://github.com/th0r3nt/AAF-Autonomous-Agent-Framework-
cd AAF-Autonomous-Agent-Framework-
```

### Шаг 2. Создание профиля агента

Запустите интерактивный менеджер AAF:

```bash
python aaf.py
```

Внутри консоли введите команду создания (например, для агента с именем `LUMI`):
```text
create LUMI
```

*Скрипт сгенерирует структуру папок `Agents/LUMI/`, настройки и подготовит Docker.*

### Шаг 3. Настройка ключей

Сверните консоль и откройте появившийся файл `Agents/LUMI/.env`. Впишите туда свои данные:
* `TG_API_ID_AGENT` и `TG_API_HASH_AGENT` (получить на https://my.telegram.org/auth).
* `LLM_API_KEY_1` (поддерживается любое количество ключей).
* Опционально: `TAVILY_API_KEY` (веб-поиск) и `OPENWEATHER_API_KEY` (погода).

Также вы можете настроить модель, таймауты и лимиты в файле `Agents/LUMI/config/settings.yaml`.

### Шаг 4. Авторизация Telegram

Возвращаемся в консоль AAF и вводим:
```text
auth LUMI
```

Скрипт попросит ввести номер телефона и код из Telegram. AAF использует **MTProto (Telethon)** - агент живет в Telegram как настоящий пользователь. Файл сессии надежно сохранится в профиле агента.

### Шаг 5. Запуск

В консоли AAF введите:
```text
start LUMI
```

Всё! Docker скачает образы, поднимет базы данных и запустит вашего агента. 
Посмотреть красивые цветные логи в реальном времени можно командой:
```text
logs LUMI
```

### 🧠 Кастомизация личности (Zero-Code)
Вы можете превратить агента в кого угодно: от сухого корпоративного аналитика до саркастичного ИИ. Не нужно знать Python или останавливать контейнеры - контекст собирается динамически.

Просто отредактируйте 3 файла в папке `Agents/<NAME>/config/personality/`:
* `SOUL.md` - ядро личности (кто он, чего хочет, как относится к миру).
* `COMMUNICATION_STYLE.md` - правила речи (использовать ли эмодзи, как обращаться к людям).
* `EXAMPLES_OF_STYLE.md` - примеры диалогов для Few-Shot промптинга.

### 🛡️ Изоляция и безопасность (True Docker-in-Docker)

AAF обеспечивает **Enterprise-уровень безопасности**, используя архитектуру True DinD (Docker-in-Docker).

Внутри стека поднимается отдельный, полностью изолированный сервис `sandbox_engine`. Главный мозг агента (`agent_lumi`) не имеет прямого доступа к Docker-демону вашего хоста. Когда агент пишет и запускает Python-скрипты, они отправляются в песочницу по внутренней сети, где для каждого скрипта создается одноразовый эфемерный микро-контейнер.

**Что это дает:**
1. **Безопасность хоста:** даже если LLM сгаллюцинирует деструктивный код (например, `rm -rf /`), он уничтожит только временный контейнер внутри песочницы. Ваша ОС в абсолютной безопасности.
2. **Чистота среды:** скрипты не оставляют мусора и зависших процессов.
3. **Безопасное делегирование:** можно поручать агенту парсить неизвестные сайты или анализировать подозрительные файлы.

### 🧩 Расширяемость: Плагины vs Песочница

AAF предоставляет два совершенно разных механизма выполнения кода, покрывая 100% сценариев.

#### 1. Песочница (Sandbox) - уровень рутины
* **Как работает:** агент *сам* пишет Python-скрипт и отправляет его в изолированный `sandbox_engine` (Docker-in-Docker).
* **Безопасность:** 100% изоляция. Скрипт не имеет доступа к ядру хоста и базам данных.
* **Применение:** делегирование математики, одноразовый парсинг "грязных" сайтов, конвертация форматов, создание фоновых демонов-наблюдателей.
* **Кто пишет код:** нейросеть (LLM).

#### 2. Плагины (Skills) - уровень мозга
* **Как работает:** вы (как разработчик) создаете `.py` файл в папке `Agents/<NAME>/plugins/`.
* **Безопасность:** 0% изоляции. Плагин работает в главном Event Loop AAF.
* **Применение:** интеграция с внешними API (Binance, AWS, Умный дом), требующая секретных ключей из `.env`. Прямые манипуляции с векторными и графовыми базами данных (`memory_manager`). Тяжелые синхронные вычисления.
* **Кто пишет код:** человек.

**Как создать свой плагин?**
Вам не нужно лезть в исходный код. Откройте автоматически сгенерированный шаблон `Agents/<NAME>/plugins/example_plugin.py`. 
Изучите пример > создайте новый файл `new_plugin_name.py` > напишите свою функцию, добавьте декоратор `@llm_skill` > (опционально) впишите нужные библиотеки в `custom_requirements.txt`. При следующем запуске (`aaf.py start <NAME>`) Docker сам скачает зависимости, а навык появится в арсенале агента.

## 🛠 Tech Stack

* **Core:** Python 3.11+, Asyncio, Pydantic, Docker, caffeine
* **Memory:** PostgreSQL (SQLAlchemy + JSONB), ChromaDB (Vector), KuzuDB (Graph)
* **Sensors:** Telethon (MTProto), psutil
* **LLM Engine:** Support for any OpenAI-compatible API (Gemini, GPT, Claude, GLM, DeepSeek, etc.).
