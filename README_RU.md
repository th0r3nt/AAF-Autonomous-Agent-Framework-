# 🌌 Autonomous Agent Framework (AAF) v1.2.0

![Python Version](https://img.shields.io/badge/Python-3.11%2B-blue)
![Docker](https://img.shields.io/badge/Docker-DinD_Ready-2496ED)
![License](https://img.shields.io/badge/License-MIT-green)
![Status](https://img.shields.io/badge/Status-Beta_v1.2.0-orange)

**AAF** — это полноценная OS-level сущность, асинхронный ИИ-агент и мульти-агентная платформа, построенная на строгой микросервисной **Event-Driven** архитектуре.

В то время как популярные решения делают ставку на легковесность, TypeScript и хранение логов в текстовых файлах, **AAF — это тяжеловесный инженерный фреймворк на Python**. Он создан для тех, кому нужна сложная графовая память, абсолютная изоляция выполняемого кода и настоящая проактивность. 

Большинство современных Open-Source агентов страдают от трех проблем: амнезия (забывают контекст через 5 шагов), зацикливание (бесконечные ReAct-петли) и ад зависимостей при установке. **AAF решает их все.**

---

## 🔥 Что нового в версии 1.2.0?

* **Real-time Interrupts (Система прерываний):** агент больше не блокируется в ReAct-цикле. Если во время "раздумий" происходит важное событие (например, сообщение в Telegram), оно мгновенно инжектируется в текущий контекст LLM, позволяя агенту менять планы на лету.
* **Microservice Deployments:** теперь агент может *самостоятельно* собирать Dockerfile и деплоить 24/7 микро-сервисы в своей песочнице, общаясь с ними через встроенный `agent_sdk`.
* **Zero-Boilerplate Skills (L0-L2 Architecture):** больше никаких ручных JSON-схем. Напишите `def` в custum_plugins, повесьте декоратор `@llm_skill`, и фреймворк сам сгенерирует L0-манифест, L1-документацию и прокинет всё в контекст агента с авто-кастом типов.
* **Agentic Mesh (Swarm 2.0):** субагенты теперь могут рекурсивно спавнить других субагентов (делегирование), передавая им контекст через новую виртуальную файловую систему (VFS) или эскалировать проблему главному агенту.

## 🧠 Ключевые особенности

* **♾️ Multi-Agent CLI Manager:** разворачивайте любое количество независимых агентов на одном сервере через удобный интерактивный терминал (`aaf.py`). Docker Compose и изолированные рабочие среды генерируются автоматически.
* **🕸️ Тройная гибридная память (SOTA):** агент не теряет контекст. Интегрирована комбинация SQL (Mental State), ChromaDB (семантика) и полноценного **GraphRAG** на базе KuzuDB (нелинейные ассоциации и интуиция 2-го уровня).
* **⚡ Асинхронная шина событий (EventBus):** мозг полностью отвязан от сенсоров. Сообщения из Telegram, системные логи и фоновые задачи падают в единую очередь приоритетов. 
* **💸 API Rotator & Smart Throttling:** встроенный менеджер ключей (Round-Robin) автоматически переключается на следующий API-ключ при достижении лимитов (ошибка 429), защищая систему от падений.
* **🛡️ Изоляция кода (True DinD):** 100% защита хост-системы. Весь сгенерированный агентом код выполняется в одноразовых изолированных Docker-контейнерах внутри песочницы. 
* **🎭 Динамическая эволюция личности:** характер агента не захардкожен. В цикле "Интроспекции" агент анализирует свои диалоги и может проактивно изменять свои черты и правила поведения (Personality Tuning), сохраняя их в БД.

## ⚙️ Установка (Quick Start)

Никаких `pip install`, конфликтов версий C++ компиляторов и ручной настройки баз данных. Проект элегантно упакован в Docker.

**Требования:** Установленный Docker Desktop и Python 3.11+ (только для запуска CLI-менеджера).

### Шаг 1. Клонирование репозитория

```bash
git clone https://github.com/th0r3nt/AAF-Autonomous-Agent-Framework-
cd AAF-Autonomous-Agent-Framework-
```

### Шаг 2. Создание профиля агента
Запустите интерактивный менеджер:

```bash
python aaf.py
```

Внутри консоли введите команду создания (например, `LUMI`):
```text
AAF > create LUMI
```

*Скрипт сгенерирует структуру папок `Agents/LUMI/`, настройки и подготовит Docker-инфраструктуру.*

### Шаг 3. Настройка ключей

Откройте появившийся файл `Agents/LUMI/.env` и впишите:
* `TG_API_ID_AGENT` и `TG_API_HASH_AGENT` (получить на [my.telegram.org](https://my.telegram.org/auth)).
* `LLM_API_KEY_1` (поддерживается любое количество ключей).

### Шаг 4. Авторизация Telegram

В консоли AAF введите:
```text
AAF > auth LUMI
```

Скрипт попросит ввести номер телефона и код. AAF использует **MTProto (Telethon)** — агент живет в Telegram как настоящий пользователь (Userbot).

### Шаг 5. Запуск и логи

```text
AAF > start LUMI
AAF > logs LUMI
```

Всё! Docker скачает образы, поднимет базы данных и запустит агента. Система WatchDog сама проследит за здоровьем модулей.

## 🛠 Tech Stack

* **Core:** Python 3.11+, Asyncio, Pydantic, Docker (DinD), caffeine.
* **Memory:** PostgreSQL (SQLAlchemy + JSONB), ChromaDB (Vector), KuzuDB (Graph).
* **Sensors:** Telethon (MTProto), psutil, Jina Reader, Tavily.
* **LLM Engine:** Любое OpenAI-совместимое API (Gemini, GPT, Claude, DeepSeek и т.д.).
