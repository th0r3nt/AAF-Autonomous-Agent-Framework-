# 🌌 Autonomous Agent Framework (AAF)

**AAF** - это полноценная OS-level сущность, асинхронный ИИ-агент и мульти-агентная платформа, построенная на микросервисной Event-Driven архитектуре.

В то время как популярные решения делают ставку на легковесность, TypeScript и хранение логов в текстовых файлах, **AAF - это тяжеловесный инженерный фреймворк на Python**. Он создан для тех, кому нужна сложная графовая память, абсолютная изоляция выполняемого кода и настоящая проактивность.

Большинство современных Open-Source агентов страдают от трех проблем: амнезия (забывают контекст через 5 шагов), зацикливание (бесконечные ReAct-петли) и ад зависимостей при установке. **AAF решает их все.**

## 🔥 Ключевые особенности

* **♾️ Multi-Agent CLI Manager:** Разворачивайте любое количество независимых агентов на одном сервере через удобный интерактивный терминал (`aaf.py`). Docker Compose генерируется автоматически.
* **🧠 Тройная гибридная память (SOTA):** Агент не теряет контекст. Комбинация SQL (жесткие правила), ChromaDB (семантика) и полноценного **GraphRAG** на базе KuzuDB (нелинейные ассоциации и интуиция).
* **⚡ Асинхронная шина событий (EventBus):** Мозг полностью отвязан от сенсоров. Сообщения из Telegram, системные логи и фоновые задачи падают в единую очередь приоритетов. 
* **💸 Бесплатная работа (API Rotator):** Встроенный менеджер ключей (Round-Robin) автоматически переключается на следующий API-ключ при достижении лимитов (ошибка 429).
* **🛡️ Изоляция кода (True DinD):** Если агент пишет код, он выполняется в одноразовых изолированных Docker-контейнерах внутри песочницы. Ваша ОС в абсолютной безопасности.

## ⚙️ Установка

Никаких `pip install`, конфликтов версий C++ компиляторов и ручной настройки баз данных. Проект упакован в Docker.

**Требования:** Установленный Docker Desktop и Python 3.11+ (только для запуска CLI-менеджера).

### Шаг 1. Клонирование репозитория

```bash
git clone https://github.com/th0r3nt/AAF-Autonomous-Agent-Framework-
cd AAF-Autonomous-Agent-Framework-
```

### Шаг 2. Создание профиля агента
Запустите интерактивный менеджер:

```bash
python aaf.py
```

Внутри консоли введите команду создания (например, `LUMI`):

```text
create LUMI
```

*Скрипт сгенерирует структуру папок `Agents/LUMI/`, настройки и подготовит Docker.*

### Шаг 3. Настройка ключей

Откройте появившийся файл `Agents/LUMI/.env` и впишите:
* `TG_API_ID_AGENT` и `TG_API_HASH_AGENT` (получить на [my.telegram.org](https://my.telegram.org/auth)).
* `LLM_API_KEY_1` (поддерживается любое количество ключей).

### Шаг 4. Авторизация Telegram

В консоли AAF введите:

```text
auth LUMI
```

Скрипт попросит ввести номер телефона и код. AAF использует **MTProto (Telethon)** — агент живет в Telegram как настоящий пользователь.

### Шаг 5. Запуск
В консоли AAF введите:

```text
start LUMI
```
Всё! Docker скачает образы, поднимет базы данных и запустит агента. Посмотреть красивые логи можно командой `logs LUMI`.

## 🧩 Расширяемость: плагины vs песочница

AAF предоставляет два механизма выполнения кода:

1. **Песочница (Sandbox) - уровень рутины.** Агент *сам* пишет Python-скрипт и отправляет его в изолированный `sandbox_engine`. 100% изоляция. Идеально для одноразового парсинга или математики.
2. **Плагины (Skills) - уровень мозга.** Вы (человек) пишете `.py` файл в папку `Agents/<NAME>/plugins/`, вешаете декоратор `@llm_skill`, и фреймворк сам генерирует JSON-схему для OpenAI Tools. 0% изоляции, прямой доступ к ядру. [Читать гайд по плагинам](docs/PLUGINS_GUIDE_RU.md).

## 🛠 Tech Stack

* **Core:** Python 3.11+, Asyncio, Pydantic, Docker, caffeine.
* **Memory:** PostgreSQL (SQLAlchemy + JSONB), ChromaDB (Vector), KuzuDB (Graph).
* **Sensors:** Telethon (MTProto), psutil.
* **LLM Engine:** Любое OpenAI-совместимое API (Gemini, GPT, Claude, DeepSeek и т.д.).
