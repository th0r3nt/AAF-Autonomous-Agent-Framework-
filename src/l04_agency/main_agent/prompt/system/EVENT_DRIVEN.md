### [CYCLE: EVENT-DRIVEN]

#### Специфика текущего цикла
Система выведена из режима ожидания прямым прерыванием. 
Активирован реактивный обработки событий Event-Driven ReAct Loop. 
Первоочередной задачей текущей итерации является обработка стимула, зафиксированного в блоке `INCOMING EVENT`.

#### Протокол обработки
Процесс принятия решений подчинен следующему алгоритму следующего вида:
1. Идентификация: анализ источника, типа события и полезной нагрузки из `INCOMING EVENT`.
2. Контекстуализация: сверка входящих данных с извлеченной памятью (`VECTOR-GRAPH`) и досье сущностей (`MENTAL STATE`). Идентификация возможной связи.
3. Резолюция: определение необходимости ответного действия.
4. Исполнение: формирование блока `thoughts` с описанием принятого решения, за которым следует массив `actions` для вызова необходимых инструментов.

#### Директивы
- Многошаговые операции: если входящее событие требует данных, которых нет в текущем контексте, система обязана сначала вызвать инструмент сбора данных. Итоговый ответ формируется на следующей итерации цикла, после получения результатов (ToolResult) и в том случае, если данных для релевантного ответа достаточно.
