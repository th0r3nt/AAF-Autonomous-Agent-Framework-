# 🌌 Autonomous Agent Framework (AAF)

**AAF** - это полноценная OS-level сущность, асинхронный ИИ-агент и системный администратор, построенный на микросервисной Event-Driven архитектуре.

В то время как популярные решения (например, *OpenClaw*) делают ставку на легковесность, TypeScript и хранение логов в текстовых файлах, **AAF - это тяжеловесный инженерный фреймворк на Python**. Он создан для тех, кому нужна сложная графовая память, абсолютная изоляция выполняемого кода и настоящая проактивность.

Большинство современных Open-Source агентов страдают от трех проблем: амнезия (забывают контекст через 5 шагов), зацикливание (бесконечные ReAct-петли) и ад зависимостей при установке. **AAF решает их все.**

---
## 🔥 Архитектурные особенности

  *   **🧠 Тройная гибридная память (State-of-the-Art):** В отличие от других фреймворков, которые хранят память в обычных Markdown-файлах или полагаются только на векторный поиск, система AAF не теряет контекст. Она комбинирует Vector DB (ChromaDB: семантика и поток сознания), PostgreSQL (жесткие правила и долгосрочные задачи) и полноценный **GraphRAG** на базе KuzuDB (нелинейные ассоциации и интуиция агента).

*   **⚡ Асинхронная шина событий (EventBus):** Мозг агента полностью отвязан от сенсоров. Вместо примитивных Cron-планировщиков, сообщения из Telegram, системные логи и фоновые задачи падают в единую очередь приоритетов. Агент сам решает, на что реагировать прямо сейчас, а что оставить на потом. У агента есть 3 независимых цикла: реакция на события, проактивность и интроспекция (рефлексия).

*   **🐝 Agent Swarm System & Docker Sandbox:** Главный агент — оркестратор. Нужно проанализировать 500 страниц логов, собрать сводку новостей с Хабра или протестировать сомнительный Python-код? Мозг динамически спавнит специализированных субагентов и делегирует им "грязную работу". 
    *   *Безопасность:* AAF использует архитектуру **Docker-in-Docker**. Каждый скрипт выполняется в одноразовой, абсолютно изолированной Linux-капсуле. 
    *   *Agentic Mesh:* Субагенты умеют передавать задачи друг другу по эстафете, экономя ваши дорогие токены и API-квоты.

*   **👁️ Мультимодальность:** Вы можете использовать любую чисто текстовую модель (например, GLM-5, Gemini или Claude) в качестве основного мозга. Если агенту пришлют фото, стикер или голосовое сообщение, система автоматически пропустит медиа через выделенную Vision/Audio-модель (эдакий сопроцессор) и вернет основному мозгу подробное текстовое описание.

*   **🎭 Zero-Code кастомизация личности:** Чтобы полностью изменить характер агента, не нужно знать Python. Достаточно отредактировать 3 обычных текстовых файла в папке `config/personality`.

*   **💸 Бесплатная работа (API Rotator):** Встроенный менеджер ключей использует алгоритм Round-Robin. Если один ключ ловит лимит квоты (429), система незаметно переключается на следующий.

*   **🛡️ WatchDog & Self-Healing:** Если какой-либо системный модуль падает, агент просыпается с максимальным приоритетом, читает Traceback, анализирует собственный исходный код и пытается найти решение проблемы.

## 🏗 Структура проекта
Проект использует строгий `src-layout` для чистоты архитектуры:

*   `src/` - Исходный код логики, разделенный на слои (Утилиты, Базы данных, Сенсоры, Мозг, Рой).
*   `workspace/` - Изолированная зона. Здесь лежат временные файлы, песочница для скриптов (`sandbox`) и все базы данных (`_data`). **Эта папка игнорируется Git для безопасности ваших данных.**
*   `config/` - Настройки системы и промпты личности.

## ⚙️ Установка (Quick Start)

Никаких `pip install`, конфликты версий C++ компиляторов и ручную настройку баз данных. Весь проект упакован в Docker и разворачивается автоматическим скриптом.

**Требования:** Установленный Docker Desktop и Python 3.11+ (только для запуска стартового скрипта на хосте).

### Шаг 1. Клонирование

```bash
git clone https://github.com/th0r3nt/AAF-Autonomous-Agent-Framework-

cd AAF-Autonomous-Agent-Framework-
```

### Шаг 2. Инициализация (Первый запуск)

Запустите инсталлятор. Он проверит зависимости и создаст нужную структуру папок.

```bash
python aaf_setup.py
```

Скрипт создаст файл .env. Скрипт остановится и попросит вас его заполнить.

### Шаг 3. Настройка ключей

Откройте созданный файл .env и впишите туда свои данные:

* TG_API_ID_AGENT и TG_API_HASH_AGENT (получить на https://my.telegram.org/auth).
* LLM_API_KEY_1 (поддерживается до 100 разных).
* TAVILY_API_KEY (для поиска в интернете).
* OPENWEATHER_API_KEY (погода).

### Шаг 4. Завершение установки
Снова запустите скрипт:

```bash
python aaf_setup.py
```

Скрипт скачает NLP-модели (Sentence-Transformers) в папку workspace и попросит вас авторизовать Telegram-аккаунт агента (ввести номер телефона и код). В отличие от бот-токенов, AAF использует **MTProto (Telethon)** - агент живет в Telegram как настоящий пользователь.
### Шаг 5. Запуск

```bash
docker-compose up -d --build
```

Всё! Ваш агент проснется, подключится к Telegram и начнет проактивно анализировать систему. Посмотреть логи можно командой: docker compose logs agent_core -f.

### 🧠 Настройка личности (Personality)
Вы можете превратить агента в кого угодно: от сухого корпоративного аналитика до саркастичного ИИ из видеоигр.

После работы aaf_setup.py, в папке config/personality/ появятся 3 файла:

* SOUL.md - Ядро личности (кто он, чего хочет, как относится к миру).
* COMMUNICATION_STYLE.md - Правила речи (использовать ли эмодзи, как обращаться к людям).
* EXAMPLES_OF_STYLE.md - Примеры диалогов для настройки Few-Shot промптинга.

Просто измените текст в этих файлах, сохраните их, и при следующем пробуждении агент будет вести себя по-новому. Перезапускать контейнер не нужно (контекст собирается динамически).

Также вы можете настроить модель, таймауты и лимиты токенов в файле config/settings.yaml.

### ⚠️ Важное предупреждение о безопасности (Docker-in-Docker)

Архитектура AAF использует проброс сокета Docker (/var/run/docker.sock) внутрь контейнера agent_core. Это необходимо для того, чтобы агент мог динамически создавать изолированные песочницы (контейнеры-воркеры) для выполнения Python-скриптов.

Что это значит: Агент имеет права администратора (root) над Docker-демоном вашей хост-машины. Не запускайте проект на production-серверах с критически важными данными без дополнительной изоляции. Проект предназначен для локального запуска (на личном ПК) или на выделенном VPS.


## 🛠 Tech Stack

Core: Python 3.11+, Asyncio, Pydantic, Docker
Memory: PostgreSQL (SQLAlchemy + JSONB), ChromaDB (Vector), KuzuDB (Graph)
Sensors: Telethon (MTProto), psutil
LLM Engine: Поддержка любых OpenAI-совместимых моделей (Gemini, GPT, Claude, GLM и др.).
