# AAF (Autonomous Agent Framework) - система, которая реализует концепцию LLM-OS.

В то время как корпорации пытаются уместить больше данных в контекстное окно LLM путем сверхдорогого масштабирования, AAF строит **полноценную оптимизированную агентскую OS**. В фреймворке AAF LLM выступает в роли CPU, который принимает решения и оркестрирует базами данных, интерфейсами и функциями.

В основе данной системы находится три типа памяти:
SQL DB - для строгих записей;
Vector DB - для семантического поиска и консолидации данных;
Graph DB - для сохранения логики и связей.

Отдельно выделяется Vector-Graph-RAG - мною написанный механизм, который позволяет искать смыслы сразу из векторов и графов, оптимизируя контекст LLM путем динамического RAG-поиска релевантной информации, не нагружая контекст мусором.

* В основе агента лежит механизм ReAct циклов. Для любого взаимодействия агент входит в ReAct итерацию: получение начальных данных > действие > получение результата действий > наблюдение > действие > ...
Агент может вызывать функции: механизм реализован через Function Calling. Для любого взаимодействия есть универсальная JSON схема:

```json
{
  "name": "execute_skill",
  "description": "Главный интерфейс для взаимодействия.",
  "parameters": {
    "type": "object",
    "properties": {
      "thoughts": {
        "type": "string",
        "description": "Цепочка рассуждений перед действием."
      },
      "actions": {
        "type": "array",
        "description": "Список действий для параллельного выполнения",
        "items": {
          "type": "object",
          "properties": {
            "tool_name": {
              "type": "string",
              "description": "Имя функции, например: vfs.files.read_file"
            },
            "parameters": {
              "type": "object",
              "description": "Аргументы функции в формате ключ-значение",
              "additionalProperties": true
            }
          },
          "required": ["tool_name", "parameters"]
        }
      }
    },
    "required": ["thoughts", "actions"]
  }
}

```

Таким образом, агент обязан перед любым действием генерировать цепочку мыслей - это предохраняет агента от необдуманных действий и позволяет сохранить детальную цепочку мыслей.
Агент может выполнять функции параллельно, вызывая сразу 1-10 функций за одну ReAct итерацию.

ReAct итераций существует три вида:
Event-Driven  ReAct Loop - мгновенный ответ на входящее событие (например, входящее личное сообщение в Telegram).
Proactivity   ReAct Loop - проактивный цикл действий, который реализует автономность агента.
Consolidation ReAct Loop - специальный цикл, который создан для очищения баз данных от устаревших/дублирующихся данных, консолидации мыслей и изученных фактов впоследствии прошлых взаимодействий.

У AAF существует слой интерфейсов (l02_interfaces) - он предоставляет отличные возможности для агента, реализуя удобное подключение/отключение к любым сервисам.
При установке пользователь может выбирать сам, какие именно сервисы (GitHub, Email, Reddit) он хочет подключить агенту, также выбирая уровень автономности - в фреймворке есть возможность дать агенту собственные аккаунты на указанных платформах.
