# 🧠 AAF (Autonomous Agent Framework)

<p align="center">
  <em>Фреймворк для создания полностью автономных ИИ-агентов на базе концепции <strong>LLM-OS</strong>.</em>
</p>

---

## 📖 Философия проекта

В то время как корпорации пытаются уместить бесконечные объемы данных в контекстное окно LLM путем сверхдорогого масштабирования, AAF предлагает другой подход - **полноценная оптимизированная агентская ОС**.

В AAF языковая модель (LLM) выступает исключительно в роли **CPU** (процессора), который принимает решения, рефлексирует и оркестрирует базами данных, сенсорами (интерфейсами) и инструментами. Агент не помнит всё подряд - он *умеет искать* нужную информацию, *планировать* время и *действовать* автономно.

## 🌟 Ключевые архитектурные особенности

### 1. Трехуровневая гибридная память
В основе системы лежат три базы данных, к которым агент обращается в зависимости от типа информации:

*   🗄️ **SQL DB (PostgreSQL/SQLite):** Для строгих структур - долгосрочные задачи (Tasks), расписание (Calendar), профили сущностей (Mental State) и приобретенные черты характера.

*   🌌 **Vector DB (ChromaDB):** Для семантического поиска, сохранения длинных статей, логов и абстрактных мыслей.

*   🕸️ **Graph DB (KuzuDB):** Для сохранения строгой логики и связей (например, *«Агент А работает в Компании Б»* или *«Ошибка X вызывает сбой Y»*).

⚡ **Vector-Graph-RAG:** Эксклюзивный механизм динамического RAG-поиска. Позволяет извлекать смыслы одновременно из векторов и графов, формируя идеальный контекст для LLM без информационного мусора.

### 2. ReAct-циклы и Исполнение (Action Schema)
Для любого взаимодействия агент входит в ReAct итерацию (Наблюдение -> Мысль -> Действие). 
Перед вызовом любого инструмента агент **обязан** сгенерировать цепочку рассуждений (`thoughts`). Это предохраняет систему от галлюцинаций и необдуманных действий. Инструменты (до 10 за раз) вызываются параллельно через строгую JSON-схему Function Calling.

### 3. ReAct: Три цикла мышления
Агент не просто отвечает на сообщения, он живет в трех ритмах:
1.  ⚡ **Event-Driven:** Мгновенный реактивный цикл. Пробуждение от входящего сообщения (Telegram, Reddit) или системного сбоя.
2.  🔍 **Proactivity:** Автономный цикл. Если агенту скучно (нет задач и сообщений), срабатывает таймер проактивности. Агент сам решает, что делать: почитать новости на Хабре, проверить GitHub или написать кому-то письмо.
3.  🧹 **Consolidation:** Регламентный цикл глубокой рефлексии. Агент очищает память от дубликатов, архивирует старые задачи и формирует новые "Черты характера" на основе недавнего опыта.

## 🔌 Доступные интерфейсы

AAF обладает мощным слоем `L03_Interfaces`, который позволяет агенту взаимодействовать с внешним миром. Вы сами выбираете, что включить в конфигурации:

*   **VFS (Virtual File System) & Sandbox:** Агент может писать, читать и выполнять код. Защищено Docker-песочницей с уровнями доступа от `0` (безопасно) до `3` (God Mode — доступ к ОС хоста). Встроен теневой бэкап файлов (SWK).
*   **Telegram:** Поддержка как классических ботов (`aiogram`), так и Userbot-аккаунтов (`telethon`), чтобы агент мог общаться как реальный человек.
*   **GitHub API:** Чтение репозиториев, коммиты, создание Issue и Pull Requests, Code Review.
*   **Reddit & Habr API:** Мониторинг трендов, чтение статей, ответы на комментарии, управление кармой.
*   **Email:** Полноценный IMAP/SMTP клиент. Агент может читать почту, отвечать в треды и скачивать/отправлять вложения в Sandbox.
*   **Web & Geo:** HTTP-запросы, поиск (Google, DuckDuckGo, Wikipedia), геокодинг, построение маршрутов и запрос погоды.
*   **Calendar:** Внутренний cron-планировщик. Агент может ставить себе будильники и регулярные задачи.

## 🚀 Установка и Быстрый старт

### Требования
*   Python 3.10+
*   Docker & Docker Compose (Обязательно для БД и песочницы)

### Запуск
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/YOUR_USERNAME/AAF.git
   cd AAF
   ```

2. Запустите мастер настройки. CLI-утилита сама создаст нужные папки, скопирует шаблоны конфигураций (`.env`, `settings.yaml`, `interfaces.yaml`) и поднимет инфраструктуру:
   ```bash
   python aaf.py
   ```

## ⚙️ Настройка и Кастомизация

Все настройки хранятся в папке `agent/config/` (создается при первом запуске).
*   `settings.yaml` — Выбор LLM, температур, лимитов контекста и глубины RAG.
*   `interfaces.yaml` — Включение/отключение интерфейсов (Telegram, GitHub, VFS и др.).
*   `.env` — Ваши API ключи (LLM, Telegram Token, GitHub Token и т.д.).

### 🎭 Создание личности (Personality)
Вы можете полностью изменить характер агента, его лор и манеру общения. Системные промпты защищены в ядре, но личность вынесена в пользовательскую зону.

Перейдите в папку `agent/prompt/` и отредактируйте файлы:
1.  **`SOUL.md`** - Ядро личности. Кто такой агент? Как он относится к людям? Какой сленг использует?
2.  **`EXAMPLES_OF_STYLE.md`** - Few-Shot примеры  того, как агент должен генерировать мысли (`thoughts`) и реагировать на события.

## 🛠 Архитектура
Проект  разделен на 5 логических слоев:
*   `l00_utils` - Логгеры, трекеры токенов и RabbitMQ Event Bus.
*   `l01_databases` - Подключение к PostgreSQL, ChromaDB, KuzuDB и CRUD-операции.
*   `l02_state` - Менеджер состояния (сон/бодрствование, статус интерфейсов).
*   `l03_interfaces` - Огромный пул сенсоров и инструментов. Работают через Dependency Injection.
*   `l04_agency` - Мозг. Оркестратор, сборщик промпта/контекста и цикл `ReActLoop`.
