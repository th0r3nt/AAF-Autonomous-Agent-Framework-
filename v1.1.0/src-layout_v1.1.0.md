🏠 AAF/
├── ⚙️ aaf.py                   <- Утилита-менеджер (CLI). Создает агентов, собирает docker-compose, смотрит логи.
├── 🐳 docker-compose.yml       <- Генерируется автоматически. Содержит базу, песочницу (DinD) и контейнеры агентов.
│
├── 📂 Agents/                  <- ИЗОЛИРОВАННЫЕ ПРОФИЛИ АГЕНТОВ
│   └── 📂 VEGA/                <- Пример профиля агента
│       ├── 🔑 .env             <- API-ключи конкретно этого агента.
│       ├── 📂 config/          
│       │   ├── ⚙️ settings.yaml          <- Настройки лимитов, LLM, таймеров.
│       │   └── 📂 personality/           <- Промпты характера (SOUL, STYLE, EXAMPLES).
│       └── 📂 workspace/                 <- Память и рабочая зона
│           ├── 🗄️ _data/                 <- Базы данных (KuzuDB, ChromaDB, файлы сессий ТГ).
│           ├── 🗑️ temp/                  <- Временные файлы (мусор очищается сам через TTL).
│           └── 🔐 sandbox/               <- Зона песочницы. Только сюда агенты могут писать Python-скрипты.
│               └── 📄 agent_sdk.py       <- Мост из песочницы обратно к агенту (шлет вебхуки).
│
└── 📂 src/                     <- ЯДРО СИСТЕМЫ (Единое для всех агентов)
    ├── 🚀 main.py              <- Точка входа в контейнере агента. Запускает все циклы.
    │
    ├── 📂 layer00_utils/       <- СЛОЙ 0: ИНФРАСТРУКТУРА И УТИЛИТЫ
    │   ├── 📄 config_manager.py     <- Парсер settings.yaml через Pydantic.
    │   ├── 📄 env_manager.py        <- Подхватывает .env нужного агента по переменной $AGENT_NAME.
    │   ├── 📄 workspace.py          <- Управление папкой temp и sandbox (очистка, защита Path Traversal).
    │   ├── 📄 logger.py             <- Цветной логгер с префиксами.
    │   ├── 📂 watchdog/             <- Нервная система. Ловит ошибки, перезапускает упавшее.
    │   └── 📂 sandbox_env/          <- Подключение к Docker-in-Docker для запуска кода.
    │
    ├── 📂 layer01_datastate/   <- СЛОЙ 1: ДАННЫЕ И СОСТОЯНИЕ
    │   ├── 📂 event_bus/            <- Асинхронная шина (publish/subscribe).
    │   ├── 📂 global_state/         <- Мониторинг времени, погоды.
    │   ├── 📂 sql_db/               <- PostgreSQL (Картина мира, задачи, логи действий и диалогов).
    │   ├── 📂 vector_db/            <- ChromaDB (Ассоциации, сырые факты, дневник интроспекции).
    │   ├── 📂 graph_db/             <- KuzuDB (Нейронные связи между сущностями).
    │   └── 📄 memory_manager.py     <- Фасад. Единая точка доступа ко всем 3-м базам данных.
    │
    ├── 📂 layer02_sensors/     <- СЛОЙ 2: ОРГАНЫ ЧУВСТВ
    │   ├── 📂 telegram/             <- Telethon. 
    │   │   ├── 📂 agent_account/    <- Авторизация и перехват событий ТГ.
    │   │   └── 📂 shared_tools/     <- API обертки (отправка, бан, опросы и т.д.).
    │   ├── 📂 pc/                   <- (Работает только если запущен локально без Headless) Микрофон, звук.
    │   └── 📄 sandbox_listener.py   <- Aiohttp-сервер (порт 18790). Слушает крики о помощи из песочницы.
    │
    ├── 📂 layer03_brain/       <- СЛОЙ 3: МОЗГ И МЫШЛЕНИЕ
    │   ├── 📄 events_monitoring.py  <- Ловит события из шины. Важные кидает в мозг, мелкие (MEDIUM/LOW) - в фон.
    │   ├── 📂 agent/
    │   │   ├── 📂 engine/           <- Оркестратор. PriorityQueue (Событие > Проактивность > Интроспекция).
    │   │   │   └── 📄 react.py      <- Тот самый цикл ReAct (сборка context -> запрос LLM -> вызов tools -> повтор).
    │   │   └── 📂 skills/           <- Навыки агента.
    │   │       ├── 📄 auto_schema.py<- Декоратор @llm_skill. Сам собирает JSON-схемы для OpenAI API.
    │   │       └── 📄 registry.py   <- Реестр, куда стягиваются все навыки.
    │   └── 📂 llm/
    │       ├── 📄 client.py         <- Ротатор API ключей. Защита от лимитов (429) и обнуление в 12:00 МСК.
    │       ├── 📂 context/          
    │       │   └── 📄 builder.py    <- СБОРЩИК RAG: Вытаскивает графы, векторы, логи и клеит Markdown.
    │       └── 📂 prompt/           <- Склейка личности агента (SOUL + STYLE + INSTRUCTIONS).
    │
    └── 📂 layer04_swarm/       <- СЛОЙ 4: РОЙ СУБАГЕНТОВ
        ├── 📄 manager.py            <- Управляет жизнью субагентов (воскрешает из БД, спавнит, убивает).
        ├── 📄 engine.py             <- Облегченный ReAct-цикл для субагентов (используют дешевые модели).
        ├── 📂 models/               <- Роли (Workers: Researcher, Analyst... / Daemons: WebMonitor).
        └── 📂 tools/                <- Спец-инструменты субагентов (Эскалация, Передача эстафеты).
